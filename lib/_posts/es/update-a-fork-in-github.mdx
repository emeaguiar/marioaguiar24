Cuando estoy contribuyendo ya sea al **core de WordPress** o a algún otro projecto
de Github, me gusta crear mi propio repositorio para no crear problemas en el original.

El problema es que muchas veces el repositorio original se actualiza antes de que
yo termine mi trabajo, creando algunos conflictos en el pull request, como esto
no existe en mi fork, siempre me veo obligado a buscar la forma de **actualizar mi
fork con el trabajo del repositorio principal**.

Esta es la forma más fácil que he encontrado:

**Paso 1**: Agregamos un nuevo `remote` con el repositorio original
(se puede llamar como queramos, generalmente “upstream”)

```bash
git remote add upstream https://github.com/usuario/repo-original.git
```

**Paso 2**: Hacemos un `fetch` con los cambios del repositorio original
(no `pull`, no queremos hacer `merge` aún)

```bash
git fetch upstream
```

**Paso 3**: Hacemos un `rebase` para reemplazar el master de nuestro repositorio,
con el del original

```bash
git rebase upstream/master
```

**Paso 4**: Hacemos `push` para tener el master actualizado en nuestro repo

```bash
git push origin master
```

Ahora podemos hacer `merge` del nuevo master en nuestro branch para resolver conflictos.
